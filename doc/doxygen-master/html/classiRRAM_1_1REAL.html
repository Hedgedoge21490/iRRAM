int size ( const REAL &  x)
friend

Returns a tight bound to the logarithmic value of |x|: \(2^{k-2}-2^e\leq|x|<2^k\), where e is x.vsize.exponent.

This is a multi-valued function.

For \(\hat x=m\cdot2^e\) = x.vsize and \(x_\varepsilon\) = x.error, this function computes \(k=\lceil\log_2(\hat x+x_\varepsilon)\rceil>\log_2|x|\) and via reiteration enforces that \(\hat x\geq x_\varepsilon+2^{k-2}\) which is equivalent to \(2^{k-2}\leq\hat x-x_\varepsilon=(m-1)\cdot2^e-x_\varepsilon+2^e \leq|x|+2^e\), the guarrantee therefore is \(2^{k-2}-2^e\leq|x|<2^k\).

Parameters
xnon-zero real number
Returns
\(k\in\mathbb Z:2^{k-2}-2^e\leq|x|<2^k\), where e is x.vsize.exponent and min_exponent <= k < MP_max
Exceptions
iRRAM_Numerical_Exception(iRRAM_underflow_error)if x is exact with value zero. If x is not exact but zero, reiterations will be performed.
See also

char **  endptr

Converts a base-10 string in scientific notation to REAL.

If \(p\) is the current working precision of iRRAM, \(k\) the number of digits in the integer part, \(z\) the number of leading zero-digits (including the fractional part) and \(g\) the exponent, and the represented value is not zero, then the result has a relative precision of roughly \((g+k-z)\log_2(10)-p\) bits, which corresponds to an absolute accuracy of \(2^p\).

Parameters
[in]spointer to base-10 string; format: WSDFE where
  • W = optional whitespace; it is discarded
  • S = optional sign
  • D = optional string of digits 0..9
  • F = optional: . followed by optional string of digits 0..9
  • E = optional: e or E followed by an optional sign followed by a string of digits 0..9
  • DF must be non-empty and not just contain .
[out]endptrwhen non-NULL, the pointer to the end of the represented number is stored in *endptr
Returns
a REAL object accurate to the current working precision.
Proof
The precision depends only on DF and E; let \(x_k\ldots x_{-n}=DF[1..|F|]\) and \(z\in\mathbb N:x[1..z]=\texttt 0^z\). To represent zero up to any accuracy, only 1 bit is required. So let \(x\neq0\) and \(b\cdot2^c\) with \(b\in[2^{-1};1)\) be the result. Then the error is

\[ (*)=|10^g\sum_{i=-n}^{k-z} x_i\cdot 10^i-2^cb| =|2^{\overbrace{(g+k-z+1)\log_2(10)}^{=:q}} \underbrace{\sum_{i=-n-(k-z+1)}^{-1}x_{i+k-z+1}\cdot10^i}_{=:x'\in[10^{-1};1)} -2^cb| \]

Now, MPFR's mpfr_strtofr() guarantees for the exact input \(2^qx'=:2^{\overline q}\overline b\) with \(\overline b\in[2^{-1};1)\) that the error is \(|2^{\overline q}\overline b-2^cb|<2^{\overline q-m}\), therefore

\[(*)=|2^qx'-2^{\overline q}\overline b+2^{\overline q}\overline b-2^cb| <|2^qx'-2^{\overline q}\overline b|+2^{\overline q-m}=2^{\overline q-m} \]

Since \(2^{\overline q}/2\leq 2^{\overline q}\overline b=2^qx'<2^q\), the error is bounded, \((*)<2^p\), when \(\overline q-p<m\leq q+1-p<\lceil(g+k-z+1)\log_2(10)\rceil+1-p\leq m'\). The exact precision is \(\max(10,m')\) where \(m'=1+\begin{cases} 1&z=k+n+1\\ \lfloor((g+k-z+1)\cdot 10+2)/3+1\rfloor-p&\text{otherwise} \end{cases}\)

double_pair iRRAM::REAL::dp

Closed double interval containg x.

Let this REAL object denote \(x\in\mathbb R\), then in this documentation dp.lower_pos is denoted as \(x_L\), dp.upper_neg is denoted as \(x_{-U}\) and likewise -dp.upper_neg is \(x_U\). This way of approximating reals is only used in the Iteration with actual_step == 1.

Invariant
value == NULL \(\Longrightarrow x_L\leq x\leq x_U\).
